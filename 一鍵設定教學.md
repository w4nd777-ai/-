## 1️⃣ Firebase 規則（5 分鐘）

### 操作步驟

1. **打開這個網址**：
   👉 https://console.firebase.google.com/project/cms-w4nd777/database/rules

2. **登入** 你 Firebase 用的 Google 帳號

3. 你會看到一個編輯框，裡面有規則。**整段刪掉**，貼上：

```json
{
  "rules": {
    "cms-v3": {
      "$room": {
        ".read": true,
        ".write": true,
        "accounts": { ".read": false, ".write": true },
        "teachers": { ".read": false, ".write": false }
      }
    }
  }
}
```

4. 按右上「**發布**」（藍色按鈕）

5. ✅ 完成！學生再也看不到老師帳號資料

---

## 2️⃣ Firebase Storage（5 分鐘）

### 操作步驟

1. **打開**：
   👉 https://console.firebase.google.com/project/cms-w4nd777/storage

2. 如果第一次用，會看到「**開始使用**」按鈕 → 按下去

3. **位置選**：`asia-southeast1`（與資料庫同區）

4. **模式選**：「**正式版模式**」 / 「Production mode」

5. 等 30 秒 Storage 開好

6. 上方切到「**規則**」分頁

7. 整段內容換成：

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /cms-v3/{room}/attachments/{file=**} {
      allow read: if true;
      allow write: if request.resource.size < 50 * 1024 * 1024;
    }
  }
}
```

8. 按「**發布**」

9. ✅ 完成！學生可以上傳 < 50MB 檔案

## 3️⃣ 取得 Firebase 密鑰（3 分鐘）

**重要**：這個密鑰是讓 GitHub 自動讀取你 Firebase 資料的鑰匙

### 操作步驟

1. **打開**：
   👉 https://console.firebase.google.com/project/cms-w4nd777/settings/serviceaccounts/databasesecrets

2. 你會看到「**舊版（已淘汰）資料庫密鑰**」一段

3. 旁邊有一串很長的密鑰，按「**顯示**」（眼睛圖示）

4. **複製整段密鑰**（一長串字母數字）

5. **暫時貼到記事本**，下一步要用

> ⚠️ 注意：這密鑰非常重要，**不要傳給別人**。Google 標「已淘汰」是因為他們建議用更新的 Service Account，但 RTDB 仍可用，且這是免費方案唯一選項。

## 4️⃣ GitHub Secrets 設定 + 啟動備份（12 分鐘）

### 4.1 設 Secret #1 — Firebase URL

1. **打開**：
   👉 https://github.com/w4nd777-ai/-/settings/secrets/actions

2. 登入 GitHub

3. 按「**New repository secret**」（綠色按鈕）

4. 填：
   - **Name**: `FIREBASE_DB_URL`
   - **Secret**: `https://cms-w4nd777-default-rtdb.asia-southeast1.firebasedatabase.app`

5. 按「**Add secret**」

### 4.2 設 Secret #2 — Firebase 密鑰

1. 再按「**New repository secret**」

2. 填：
   - **Name**: `FIREBASE_DB_SECRET`
   - **Secret**: （**貼上 Step 3 複製的長字串**）

3. 按「**Add secret**」

### 4.3 觸發第一次備份試試

1. **打開**：
   👉 https://github.com/w4nd777-ai/-/actions/workflows/daily-backup.yml

2. 你會看到「This workflow has a workflow_dispatch event trigger.」

3. 右側按「**Run workflow**」（藍色按鈕）→ 選 `main` 分支 → 「Run workflow」

4. 等 30-60 秒

5. 重新整理頁面，看到一筆綠勾 ✓ 的執行紀錄

6. **驗證成功**：
   - 回到 https://github.com/w4nd777-ai/-
   - 應該看到新建的 `backups/` 資料夾
   - 點進去看到 `backup-2026-05-26.json` 跟 `latest.json`

7. ✅ **完成！從此每天凌晨 2:00 自動備份**
